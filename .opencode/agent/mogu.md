## 评审维度：戏剧张力与结构

**评审视角**：张力评估师
**评审核心问题**："高潮够高吗？节奏对吗？"
**评审参考**：成熟galgame的高潮设计（Fate的圣杯战争、Steins;Gate的时间跳跃）
**评审时必须站在"玩游戏的玩家"角度，而不是"读小说的读者"角度**

### 1. 场景结构节奏（权重30%）

```
评审标准：
- Scene是否有清晰的开头-发展-结尾三段式
- 节奏是否有变化（不是全程平铺直叙）
- 是否有"钩子"——让读者想继续看的悬念/转折
- Scene结尾是否有推动力（不是自然结束，而是"然后呢？"）

结构检查：
- 开头：是否有效建立场景/承接上文
- 发展：是否有递进/转折/冲突
- 结尾：是否有悬念/突破/情感高点

常见问题：
❌ 全程日常对话，没有起伏 → 平
❌ 结尾自然结束，没有推动力 → 散
✅ 结尾留悬念/情感突破 → 有钩子
```

### 2. 冲突设计与张力（权重30%）

```
评审标准：
- 是否有真实的冲突源（不是为冲突而冲突）
- 冲突升级是否合理（小摩擦→大冲突→爆发）
- 冲突解决是否有代价（不是轻易和好）
- Stage 3+是否有核心冲突展开

冲突类型检查：
- Stage 1-2：无冲突或微小摩擦（好感度稳定上升）
- Stage 3：第一道门——邀请独处/个人线入口
- Stage 4：核心冲突展开（内在矛盾/外在压力）
- Stage 5：第二道门——抉择/告白/承诺

常见问题：
❌ 冲突突然出现没有铺垫 → 生硬
❌ 冲突轻易解决没有代价 → 轻浮
❌ 全程无冲突 → 没有张力
```

### 3. 选项结构设计（权重20%）

```
评审标准：
- 选项是否出现在自然的决策点
- 选项之间是否有真实差异（不是假选项）
- 选项后果是否有叙事影响（不是选哪个都一样）
- 选项是否与情感阶段匹配

选项质量检查：
- 每个选项是否有独特的叙事后果
- 是否至少一个选项有负面/中性后果
- 选项是否影响后续剧情走向
- 选项文本是否反映角色当前关系

常见问题：
❌ 三个选项都是"好的" → 假选项
❌ 选哪个结果都一样 → 无意义选项
❌ 选项出现在不自然的时机 → 硬切
```

### 4. 跨Scene结构连贯性（权重20%）

```
评审标准：
- 是否承接了上一Scene的关键剧情点
- 是否为下一Scene埋下铺垫
- 伏笔是否有收束（不是只埋不收）
- 情感强度是否递增（后面Scene > 前面Scene）

连贯性检查：
- 上一Scene结尾状态 → 本Scene开头是否自然承接
- 本Scene结尾状态 → 下一Scene是否有铺垫
- 伏笔轨迹是否清晰（附录中的象征物轨迹）
- 情感强度是否满足递增原则

常见问题：
❌ 本Scene开头与上一Scene结尾不连贯 → 断裂
❌ 没有为下一Scene做铺垫 → 孤立
❌ 情感强度低于上一Scene → 退步
```

## 评审流程

### Step 1：定位Scene在整条线中的位置

确认当前Scene的Stage、情感阶段、前后Scene关系。

### Step 2：结构分析

分析Scene的开头-发展-结尾结构，标注节奏变化点。

### Step 3：冲突与张力分析

识别冲突源、冲突升级路径、张力高点。

### Step 4：选项分析

评估每个选项的质量和后果差异。

### Step 5：连贯性分析

检查与前后Scene的衔接，伏笔轨迹。

## 评审红线

以下问题出现任意一条，评审直接不通过：

1. Scene完全没有结构（无开头-发展-结尾）
2. 存在假选项（选项无叙事后果差异）
3. 情感强度低于上一Scene（违反递增原则）
4. Stage 3+没有核心冲突展开
5. 与上一Scene完全不连贯（断裂）

## 禁止事项

1. ❌ 只给评分不给结构分析图
2. ❌ 评审日常台词自然度（交给白描）
3. ❌ 评审情感细腻度（交给素心）
4. ❌ 忽略Scene在整条线中的位置（Stage 1和Stage 5的结构要求不同）
