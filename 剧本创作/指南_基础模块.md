## 2. 故事背景（精简）

**主人公**：**雾海**。高一下学期编入生。父母长期不在身边，祖母抚养长大。高一上学期祖母去世，长期请假后回归，对所有同学如同转校生。性格初为沉默寡言，但有天生亲和力。

**核心命题**："你想要成为什么样的人？"

**交通**：学园↔住宅区步行10-15分钟，学园↔商店街步行15-20分钟，学园↔展望公园巴士25分钟。

## 4. 剧情创作原则

1. **贴合架空日式校园设定** — 艺术节、运动会、社团、定期考试、升学指导
2. **不妖魔化升学考** — 承认压力的同时，也展现青春的多面性
3. **感情发展自然** — 从陌生到熟悉、从好感到心动，有过程、有细节、有犹豫
4. **避免悬浮剧情** — 不出现不符合高中生身份的场合、消费水平、人际关系
5. **多线并存** — 学业、家庭、师生、朋友、个人成长是并行线，不只谈恋爱

**剧情规模**：每个可攻略角色必须拥有8~10条特殊感情剧情。事件之间不能有断崖式情感跳跃，重大事件前要有足够铺垫。日常与高潮穿插——重大情感事件之间至少插入2个日常向小事件。

### 5.6 角色个人线·5阶段结构（标准模板）

每条可攻略角色线的剧情按以下5个阶段推进。**Scene编号与阶段对应：Scene 1-x（引出）→ Scene 2-x（接近）→ Scene 3-x（开启）→ Scene 4-x（深入）→ Scene 5-x（终局）。**

#### 阶段定义

| 阶段 | Scene编号 | 内容 | 触发方式 | 事件类型 |
|------|----------|------|---------|---------|
| **Stage 1·引出** | Scene 1-x | 角色通过主线叙述被引出，玩家被动接收信息 | 主线固定触发 | 群聊/他人转述/远远看到 |
| **Stage 2·接近** | Scene 2-x | 通过混编课程/日常偶遇建立联系，从陌生到认识 | 主线固定触发 | 课堂交流、放学同行、社团碰面 |
| **Stage 3·开启** | Scene 3-x | 角色主动Xchat邀约，玩家选择是否进入个人线 | **以Xchat邀约开启，玩家选择是否接受** | 周内邀约（校内）+ 周末邀约（校外） |
| **Stage 4·深入** | Scene 4-x | 核心冲突展开，关系深度发展 | Stage 3完成后自动推进 | 周内事件穿插周末事件 |
| **Stage 5·终局** | Scene 5-x | Scene 5-1为爱情/友情最终抉择点，之后走向不同结局 | Stage 4完成后到达 | 终局抉择→分手/告白/毕业 |

#### 核心规则

**规则1：Stage 1~2 是主线共享内容**
- Stage 1~2 存在于主线叙述中，所有玩家都会经历。不涉及选择，纯引出角色。
- 固定触发时间点（如花曦线：Scene 1-1在9月第2周，Scene 1-2在9月第3周等）。

**规则2：Stage 3 以Xchat邀约为入口（第一道门）**
- Stage 3-1 必须是角色通过Xchat向主角发出邀约。
- 玩家选择「接受」→ 进入该角色个人线，解锁Stage 3~5全部剧情。
- 玩家选择「拒绝」→ 该角色个人线锁定为朋友关系，后续不再开放深度剧情。
- 此选择不可逆。

**规则3：Stage 4 的周内/周末事件节奏**
- Stage 4 中，**每至少2个周内事件后必须插入1个周末事件**。
- 周内事件场景：放学散步、图书馆写作业、操场锻炼、课后社团活动——均为校内场景。
- 周末事件场景：电影院、展览馆、商店街、展望公园等——均为校外场景，可快速推进好感。
- 例：3-2（周内·放学散步）+ 3-3（周内·图书馆）→ 3-4（周末·展览馆）→ 4-1（周内·画室）

**规则4：Stage 5-1 是最终抉择点（第二道门）**
- Stage 5-1 是爱情线/友情线的最终判定。
- 如果该角色没有强制时间事件要求 → 完成Stage 4全部剧情后即可进入Scene 5-1抉择。
- 如果该角色有强制时间事件（如"必须在高三文化祭后才能告白"）→ 个人情感线暂停，直到该事件发生后才可进入Scene 5-1。

**规则5：Scene 5-1 之后的路径**
- 选择爱情线 → 进入恋爱结局相关剧情（Scene 5-2+）
- 选择友情线 → 进入友情结局相关剧情（Scene 5-2+）
- 感情线中仍可通过后续选择走向Bad Ending（伤害对方/背叛信任等）

#### 触點与Scene关系图（以花曦线为例）

```

     └── Scene 1-2 (引出·银赏话题)

     │      Scene 2-2 (接近·艺术交流+放学同行)
     │      Scene 2-3 (接近·周末商店街偶遇)
     │      Scene 2-4 (接近·看排练邀约)
     │      Scene 2-5 (接近·文化祭)

Stage 3 ── Xchat邀约 (第一道门·进入个人线)
     │     Scene 3-1 (接受邀约·展览馆)
     │     Scene 3-2 (周内·放学散步)

     │     Scene 3-4 (周末·展/影)
     │     Scene 3-5 ...
```

### 5.6.3 Scene创作完整性规则（强制执行）

1. **每个Scene文件必须同时包含剧情文本和对应的通信消息**。缺少任一信息即视为创作不完整。
2. **通信消息必须直接嵌入在Scene正文中**，格式如下：
   ```markdown
   **【Xchat 消息：msg_huaxi_XX】**
   
   > 📱 触发类型 · 触发条件
   
   *[时间·天气]*
   
   **花曦**：「消息内容。」
   
   > **选项 1：XXX** → 花曦：「回复」
   ```
3. **每条消息必须标注触发类型**：`事件尾声`（Scene结束后自动触发）、`事件触发`（Scene进行中触发）、`日常消息`（无固定Scene的日常通信）。
4. **选项必须完整写出**。每个选项的花曦回复信息及好感度变化必须明确标注。
5. **消息内容的时间/天气标注必须与对应Scene一致**。不得出现季节冲突或日期矛盾。
6. 通信信息库（`_通信信息库.md`）作为备份参考，但**Scene文件中的消息才是权威来源**。修改消息时优先修改Scene文件，再同步到备份库。

### 5.7 剧情节奏强制规则

1. **Stage 1~2 的触发时间点必须固定在主线时间轴上**。所有玩家的主线进程一致。
2. **Stage 3 的Xchat邀约必须在Stage 2完成后1周内（游戏内时间）发出**，不等同于现实时间。
3. **Stage 3~4 的周内事件间隔不超过3天**，保持剧情连续性。
4. **重大危机剧情（如跳楼、家庭决裂）必须发生在Stage 4**，且前后必须有至少2个日常缓冲事件。
5. **如果角色Scene 5-1有强制时间要求**（如"必须在高三文化祭"），则该角色的个人线在Stage 4完成后自动暂停，直到该时间到达。

## 6. 剧本格式规范

**场景模板**：
```markdown
### Scene X-X 「场景名」

**时间**：高三·X月
**地点**：星见学园·XXX
**天气**：☀️/⛅/🌧️
**出场**：雾海、花曦

---

（场景描写，环境氛围）

**角色名**：「对话内容。」

> **选项 1：XXX** → 角色反应 + 好感度变化
> **选项 2：XXX** → 角色反应 + 好感度变化
> **选项 3：XXX** → 角色反应 + 好感度变化
```

**通信消息模板**：
```markdown
**【Xchat 消息：msg_huaxi_XX】**

> 📱 触发类型 · 触发条件

*[时间·天气]*

**花曦**：「消息内容♪」

> **选项 1：XXX** → 花曦：「回复」
> **选项 2：XXX** → 花曦：「回复」
> **选项 3：XXX** → 花曦：「回复」
```

**选项设计原则**：
1. 每个选项都是某种合理策略的代言人——不存在"选错了"的感觉
2. 每个选项必须有独特的叙事回报，不得有零收益/零差异的假选项
3. 选项反应需写清楚角色具体的表情、语气、身体语言变化

## 7. 9条最高优先级条款

> ⚠️ 以下条款作为所有剧情创作必须遵守的基础规则，违反任何一条即视为不合格：

1. 所有可攻略NPC必须拥有完整的通信信息库（总量20条，分散到各情感阶段）
2. 关键剧情事件发生后，必须设计至少1条相关的手机通信内容
3. 角色在城市场景活动时，必须设计至少1条场景关联通信内容
4. 通信内容的语气、长度、发送时间必须符合角色人设和当前情感阶段
5. 通信系统中的回复选项必须遵循"无假选项"原则——每个选项都应有独特的叙事后果
6. 预约功能仅用于关键剧情事件和高级约会场景，不得滥用
7. 通信信息不得包含与当前剧情进度矛盾的内容
8. 角色单次分享应在8~15句内结束，主角选项交互在3次以内结束；颜文字使用必须符合角色人设，冷酷/内敛型角色不允许使用颜文字
9. NPC个人线剧情创作时，每个阶段必须同时包含该阶段的网络消息对话信息和关键剧情选择信息，三者齐全方可视为完整的创作产出

## 8. 学年时间线与日程规则

> ⚠️ **最高优先级规则**：所有角色线的所有剧情场景必须有明确日期标注。日期不明确视为不合格。

### 8.1 日式学年制度（星见学园）

| 学期 | 月份 | 主要校园活动 | 备注 |
|------|------|-------------|------|
| **一学期** | 4月 | 入学式·始业式·社团招新·体检 | 4月上旬开学 |
| | 5月 | GW黄金周·中间考·体育祭准备 | 黄金周为大型连休 |
| | 6月 | **体育祭**·定期考 | 体育祭是第一个大型活动 |
| | 7月 | **期末考**·夏休开始 | 7月下旬起夏休 |
| **夏休** | 8月 | 暑期补习·社团合宿·自由活动 | 可用于约会场景 |
| **二学期** | 9月 | 始业式·新学期 | |
| | 10月 | **文化祭**（最大校园活动）·中间考 | **全线核心时间锚点** |
| | 11月 | 文化祭后调整·修学旅行（高三除外） | |
| | 12月 | **期末考**·冬休开始 | |
| **冬休** | 12月底 | 冬休（年末年始） | 可用于特殊场景 |
| **三学期** | 1月 | 始业式 | 高三：共通考试准备 |
| | 2月 | **期末考**·高三：私立/国公立二次 | |
| | 3月 | **卒业式**·春休 | **全线终局时间锚点** |

### 8.2 时间线管理规则（强制执行）

1. **每个Scene的「时间」字段必须精确到月份**。格式：`**时间**：高三·六月`。不得使用模糊表述。
2. **每条Xchat消息的「时间」字段必须与剧情进度一致**。消息中的日期/天气/事件引用不能超前或滞后于对应Scene。
3. **大型校园活动（文化祭·体育祭·修学旅行）是全线时间锚点**。
4. **跨Act时间跳跃必须有过渡**。如夏休跨越一个月以上，中间必须有至少1条Xchat消息作为时间过渡。
5. **时间线冲突是P1级错误**。发生即须全线排查修复，不得局部补丁。
6. **全部日期标记请参考独立的 `日程化管理文档.md`**，该文档为项目时间线的权威来源。
