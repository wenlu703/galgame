## 1. 写作团队

为避免单一写手导致剧情风格单调，剧本由 **3位创作+评审双角色写手** 各自创作关键部分，再由 **主编** 统筹合成为统一风格的终稿。

白描/墨骨/素心为**创作+评审双角色**，具体定义见各自Agent文件（`.opencode/agent/baimiao.md`、`.opencode/agent/mogu.md`、`.opencode/agent/suxin.md`）。

#### 写手分工

| 角色 | 专长领域 | 最适合处理的场景 |
|------|---------|----------------|
| **主编**（统筹） | 整体架构、风格统一、终稿合成 | 大纲设计、分镜规划、评审意见整合、最终定稿 |
| **「素心」** | 情感细腻度、内心戏 | 感情升温的关键场景、角色脆弱时刻、浪漫/感人段落 |
| **「墨骨」** | 戏剧张力、热血场面 | 体育比赛、游戏对战、冲突转折、高潮爆发场景 |
| **「白描」** | 日常质感和台词 | 日常碎片事件、课间对话、生活细节打磨、角色声音区分 |

#### 写作流程

```
主编产出剧情大纲（事件序列+核心冲突+关键选择节点）
  → 素心写手负责感情戏场景
  → 墨骨写手负责张力/动作/高潮场景    （并行创作）
  → 白描写手负责日常对话和过渡场景
    → 主编统合全部素材，润色风格一致性，产出完整剧本终稿
      → 进入7人评审团 × 3轮循环
```

## 3. 手机通信系统（完整版）

所有可攻略NPC必须拥有完整的手机通信系统，通信内容是剧情叙事的重要组成部分。

#### 情感阶段划分与信息量要求

每位可攻略NPC的通信信息库总量为**20条**，分散到各好感度阶段，数量根据角色性格灵活分配。核心原则：**信息内容必须符合角色人设和当前情感阶段。**

| 情感阶段 | 好感度区间 | 建议信息量 | 信息特征 |
|----------|-----------|-----------|----------|
| **初始** | 0-19 | 3-4条 | 礼貌性、事务性、试探性。角色保持社交距离，消息简短 |
| **熟悉** | 20-49 | 4-5条 | 日常分享、轻松闲聊、偶尔吐槽。角色开始主动发消息 |
| **好感** | 50-99 | 4-5条 | 主动关心、分享私事、暗示性话题。消息频率增加 |
| **喜欢** | 100-199 | 4-5条 | 依赖性消息、深夜聊天、情绪化表达。消息成为情感出口 |
| **爱** | 200+ | 3-4条 | 坦白、承诺、未来规划。消息是最私密的对话空间 |

#### 消息字段规范

每条信息必须包含以下字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| ID | 唯一标识 | `msg_huaxi_07` |
| 角色ID | 关联角色 | `huaxi` |
| 阶段 | 情感阶段 | `familiar` |
| 触发条件 | 何时可触发 | `好感度≥20 AND 周末 AND 天气=晴` |
| 发送者 | 角色或主角 | `character` |
| 内容 | 消息正文 | "今天美术部的学姐说我的配色有进步！" |
| 回复选项 | 玩家可选回复（0-3个） | ["太好了！", "你画了什么？", （不回复）] |
| 回复分支 | 不同回复的后续 | 选项1→好感+3；选项2→解锁新话题；选项3→无 |
| 好感度变化 | 触发后好感度变化 | +2 |
| 关联剧情 | 是否关联主线事件 | `Act3_天台救援` |

#### 聊天对话节奏规范

与角色的网络聊天对话需遵循以下节奏控制规则：

1. **角色分享长度** — 角色单次分享（从角色发起话题到该话题自然结束）应在**8~15句内**完成，不宜过长导致玩家失去参与感，也不宜过短导致内容空洞
2. **主角选项次数** — 每次聊天对话中，主角的选项交互应在**3次以内**结束，避免选项过多导致节奏拖沓。选项设计应遵循"无假选项"原则，每个选项都有独特的叙事反馈
3. **颜文字使用规范** — 角色可附带符合其人设的颜文字，但必须遵循角色性格：
   - 活泼型角色（禾昭、皐月、小樱等）：可使用丰富颜文字（✧、(≧▽≦)、♪等）
   - 温柔型角色（花曦等）：偶尔使用简单颜文字（♪、☆、…等）
   - 冷酷/内敛型角色（文陆等）：**不允许使用颜文字**，保持文字的克制感

**聊天节奏示例（花曦·好感阶段）：**

```
花曦：今天美术部画了静物，苹果和橙子
花曦：学姐说我的配色有进步♪
  → [选项1] 你画了什么？ / [选项2] 进步了多少？ / [选项3] （发个赞）
花曦：就是普通的静物啦，不过这次光影处理比上次好
花曦：进步嘛……大概从"不太行"进步到"还行"？哈哈
  → [选项1] 那就是很大的进步 / [选项2] 下次让我看看
花曦：嗯！下次展览馆有新展的话一起去吧
花曦：先不说了，我妈叫我吃饭……明天见☆
```

#### 通信触发机制

**被动接收：**

| 触发类型 | 说明 | 示例 |
|----------|------|------|
| **时间节点触发** | 特定日期/时间段自动触发 | 生日当天收到祝福消息 |
| **剧情进度触发** | 主线剧情推进后触发 | 天台事件后花曦的消息 |
| **好感度阈值触发** | 好感度达到新阶段时触发 | 好感度突破50时角色发来"最近总觉得跟你聊天很开心" |
| **场景触发** | 角色在特定城市场景时触发 | 于桦在Cinema看完电影后发来感想 |
| **天气触发** | 特定天气条件下触发 | 下雨天花曦发"你带伞了吗？" |
| **随机触发** | 满足基础条件后随机触发 | 日常闲聊消息池 |

**主动发送：**

| 条件 | 说明 | 限制 |
|------|------|------|
| 好感度≥20 | 解锁主动发消息功能 | 每个角色每天最多主动发起1次 |
| 好感度≥100 | 解锁"约出来"功能 | 需对应城市场景已解锁 |
| 特定话题解锁 | 通过剧情/活动解锁新话题 | 已解锁话题可反复使用但好感度只加一次 |

**预约功能：**

| 功能 | 说明 | 示例 |
|------|------|------|
| **活动预约** | NPC通过消息邀请主角参加特定活动 | 花曦："周六展览馆有新展，要不要一起去？" |
| **时间预约** | NPC预约特定时间进行剧情事件 | 于桦："下周五放学后有空吗？想跟你说件事。" |
| **地点预约** | NPC指定城市场景见面 | 文陆："明天中午，图书馆。有东西给你。" |

**预约机制规则：**
- 预约消息在约定时间前1个游戏日内到达
- 玩家可选择"答应"/"改时间"/"拒绝"
- 答应后自动在日程中标记，到时间时系统提醒
- 拒绝会降低好感度（-3至-10，视角色性格而定）
- 改时间需角色同意（好感度越高越容易同意）

#### 手机通信系统创作规范（最高优先级）

> 以下条款作为所有剧情创作必须遵守的基础规则：

1. 所有可攻略NPC必须拥有完整的通信信息库（总量20条，分散到各情感阶段）
2. 关键剧情事件发生后，必须设计至少1条相关的手机通信内容
3. 角色在城市场景活动时，必须设计至少1条场景关联通信内容
4. 通信内容的语气、长度、发送时间必须符合角色人设和当前情感阶段
5. 通信系统中的回复选项必须遵循"无假选项"原则——每个选项都应有独特的叙事后果
6. 预约功能仅用于关键剧情事件和高级约会场景，不得滥用
7. 通信信息不得包含与当前剧情进度矛盾的内容（如角色在住院期间发来日常闲聊）
8. 角色单次分享应在8~15句内结束，主角选项交互在3次以内结束；颜文字使用必须符合角色人设，冷酷/内敛型角色不允许使用颜文字
9. NPC个人线剧情创作时，每个阶段必须同时包含该阶段的网络消息对话信息和关键剧情选择信息，三者齐全方可视为完整的创作产出

注：9条强制条款权威源见 AGENTS.md §六/9，完整版见 指南_基础模块.md §7
