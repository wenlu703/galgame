### 协作机制

```
用户需求 → 总编拆解 → 分配专业Agent → Agent执行 → 总编审核 → 同步文件 → 更新任务管理
```

### 剧本创作全流程

```
Phase 1：架构设计（仅关键节点）
  Stage转换Scene/高潮Scene → 剧本架构师输出精简蓝图 → 总编审核蓝图
  复杂Scene → 可选输出节拍表（叙事节奏骨架）
  日常Scene → 剧本架构师内部检查后直接创作

Phase 2：创作执行（三层记忆+Lorebook）
  剧本架构师 → 加载三层记忆（核心/工作/归档）
    → 内部检查（静默模式）
    → 创作（Lorebook关键词触发按需注入设定）
    → 输出Scene正文（含情感弧线数值+伏笔标记）

Phase 3：三维度评审
  白描(日常真实感) + 墨骨(戏剧张力) + 素心(情感细腻度) → 各输出评审报告

Phase 4：评审汇总与决策（8分制+情感弧线验证）
  总编汇总三份报告 → 验证情感弧线数值递增
  综合评分≥8.0且三维度均≥7.0 → 通过
  综合6.0-7.9或单维度5.0-6.9 → 修改后重审（最多2轮）
  综合<6.0或单维度<5.0 → 打回重写
  情感峰值不递增 → 打回重写

Phase 5：美术需求提取
  剧本-美术衔接师 → 提取美术需求 + AI提示词 → 输出Scene美术需求单

Phase 6：文件同步
  Scene文件 + 完整版 + 附录（含伏笔状态表） + 任务管理.md + 质量趋势表 → 全部同步
```

### Agent扩展方法

1. 在 `.opencode/agent/` 下创建 `{agent-name}.md`
2. 文件开头添加YAML front matter：`description`（功能描述）、`mode: subagent`
3. 正文包含：项目上下文、必读文件、职责边界、专业规范、输出格式
4. 在本节Agent清单中登记
5. 更新 `chief-editor.md` 中的调度表

## 二、Scene阶段拆分规范

| 阶段 | 内容 | 文件 |
|------|------|------|
| Stage 1 | 引出——角色在主线中被引出 | `_Scene1.md` |
| Stage 2 | 接近——混编课/偶遇建立联系 | `_Scene2.md` |
| Stage 3 | 开启——Xchat邀约，个人线入口（第一道门） | `_Scene3.md` |
| Stage 4 | 深入——核心冲突展开 | `_Scene4.md` |
| Stage 5 | 终局——Scene 5-1抉择（第二道门） | `_Scene5.md` |
| — | 附录——选项表/象征物轨迹 | `_附录.md` |

**拆分规则**：
- 每个Scene文件开头加版本头：`> {角色名}篇 v{X} · {阶段名} / 完整版见：{角色名}篇_v{X}.md`
- 修改时必须同步更新完整版和对应Scene文件
- Scene文件中的通信消息即权威来源，不再单独维护通信备份库

## 三、创作工作流

```
Step 1：读 剧本创作/指南_基础模块.md + 目标角色文件
Step 2：读 剧本创作/任务管理.md 确认当前任务
Step 3：只加载目标Scene文件（禁止跨Stage加载）
Step 4：执行创作/修改
Step 5：同步完整版 + 标记任务✅
```

### 禁止事项

1. ❌ 禁止一次性读取全部创作指南+完整版剧本
2. ❌ 禁止跨Stage修改（改Scene2时不加载Scene3）
3. ❌ 禁止忽略任务管理
4. ❌ 禁止无结构创作

## 四、角色人设红线

详见各 `剧本创作/指南_角色_{名}.md`。一句话：
- 文陆：高冷惜字·冷面幽默·**禁止颜文字**
- 禾昭：社牛话多·感叹号多·丰富颜文字
- 花曦：活泼外壳vs脆弱内核·**Scene 3+才有颜文字**·扣子只在主角面前独处时解
- 于桦：温和外壳下的孤独·偶尔颜文字

---

## 五、最高指令

> ⚠️ **Scene 3-1 起，该角色个人线中禁止出现其他可攻略角色。**
> 只保留当前角色 + 主角 + 无关NPC。
> 除非主编强制要求，否则文陆/禾昭/于桦不得在花曦Scene 3+中出现。

---

## 六、通信消息规范（9条）

1. 每角色20条消息分散到各情感阶段
2. 关键事件后必有1条通信
3. 城市场景活动时必有1条通信
4. 语气/长度/时间符合人设和情感阶段
5. 无假选项——每个选项有独特后果
6. 预约功能不滥用
7. 通信内容不与剧情进度矛盾
8. 单次8~15句，选项≤3次；冷酷型角色禁止颜文字
9. 消息直接嵌入Scene正文，不维护独立通信备份
